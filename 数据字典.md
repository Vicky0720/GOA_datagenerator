# 数据字典

本文档基于当前数据生成器结构，以及示例输出目录 `config_test_data/` 的实际结果整理。

## 表清单

- `users.csv`：用户主表
- `events.csv`：用户行为事件表
- `payments.csv`：支付表
- `campaigns.csv`：活动计划表
- `event_calendar.csv`：特殊事件影响表

## 1. users.csv

| 字段名 | 类型 | 示例 | 含义 | 备注 |
|---|---|---|---|---|
| `user_id` | string | `U00001` | 用户唯一标识 | 主键 |
| `register_date` | datetime | `2025-12-02 01:58:20` | 注册时间 | 在观察期开始和注册截止日之间随机生成 |
| `country` | string | `SG` | 国家/地区 | 当前枚举：`SG` `MY` `ID` `PH` |
| `device` | string | `android` | 设备平台 | 当前枚举：`ios` `android` |
| `channel` | string | `ads` | 获客渠道 | 当前枚举：`organic` `ads` `referral` `influencer` |
| `persona` | string | `casual` | 生成器内部画像标签 | 仅用于数据生成，不允许进入后续分析、展示、报告、RAG、策略推荐或 Agent 输出 |

## 2. events.csv

| 字段名 | 类型 | 示例 | 含义 | 备注 |
|---|---|---|---|---|
| `user_id` | string | `U00001` | 用户唯一标识 | 外键，关联 `users.user_id` |
| `event_time` | datetime | `2025-12-05 09:18:11` | 事件发生时间 | 按用户逐日活跃后生成 |
| `event_type` | string | `login` | 事件类型 | 当前枚举：`login` `play` `mission_complete` `level_up` `quit` |
| `session_length` | integer | `15` | 会话时长 | 单位视为分钟，最小值为 1 |

## 3. payments.csv

| 字段名 | 类型 | 示例 | 含义 | 备注 |
|---|---|---|---|---|
| `user_id` | string | `U00005` | 用户唯一标识 | 外键，关联 `users.user_id` |
| `payment_time` | datetime | `2026-01-02 23:57:52` | 支付时间 | 通常从用户已有事件时间中抽样 |
| `amount` | decimal | `19.5` | 支付金额 | 受画像、国家、活动等因素影响生成 |
| `product_type` | string | `gem_pack` | 商品类型 | 当前枚举：`starter_bundle` `battle_pass` `gem_pack` `vip_bundle` |

## 4. campaigns.csv

| 字段名 | 类型 | 示例 | 含义 | 备注 |
|---|---|---|---|---|
| `campaign_id` | string | `C001` | 活动 ID | 主键 |
| `start_date` | date | `2026-01-05` | 活动开始日期 | 固定配置生成 |
| `end_date` | date | `2026-01-12` | 活动结束日期 | 固定配置生成 |
| `target_segment` | string | `new_user` | 目标人群 | 业务标签字段 |
| `reward` | string | `starter_mission_pack` | 活动奖励 | 业务标签字段 |

## 5. event_calendar.csv

| 字段名 | 类型 | 示例 | 含义 | 备注 |
|---|---|---|---|---|
| `event_name` | string | `festival_event` | 特殊事件名称 | 来自生成器配置 |
| `start_date` | date | `2026-01-10` | 生效开始日期 | |
| `end_date` | date | `2026-01-14` | 生效结束日期 | |
| `activity_mult` | decimal | `1.15` | 活跃系数 | 大于 1 提升活跃，小于 1 降低活跃 |
| `payment_mult` | decimal | `1.1` | 付费系数 | 大于 1 提升付费 |
| `session_mult` | decimal | `1.08` | 时长系数 | 影响 `session_length` |
| `quit_mult` | decimal | `0.9` | 退出概率系数 | 影响 `quit` 事件概率 |
| `target_personas` | string | `engaged_nonpayer` / `all` | 目标画像 | 多值时用逗号分隔 |
| `target_channels` | string | `all` | 目标渠道 | 多值时用逗号分隔 |
| `target_countries` | string | `all` | 目标国家 | 多值时用逗号分隔 |

## 主键与关联

- `users.user_id` 是用户主键
- `events.user_id` 关联 `users.user_id`
- `payments.user_id` 关联 `users.user_id`
- `campaigns` 和 `event_calendar` 当前是独立配置表，不直接通过主键关联用户表

## 当前枚举值

- `persona`：`whale` `engaged_nonpayer` `casual` `at_risk`
- `country`：`SG` `MY` `ID` `PH`
- `device`：`ios` `android`
- `channel`：`organic` `ads` `referral` `influencer`
- `event_type`：`login` `play` `mission_complete` `level_up` `quit`
- `product_type`：`starter_bundle` `battle_pass` `gem_pack` `vip_bundle`

## 使用边界约定

- `users.csv` 中的 `persona` 字段仅用于数据生成阶段。
- 在 GameOps Agent 项目中，`persona` 必须视为不可见字段。
- 后续数据加载、指标计算、分群、评分、RAG、策略推荐、UI 展示、报告输出中，均不得使用 `persona` 作为输入或展示内容。
- 如果生成数据中保留了该字段，加载器应在进入业务分析链路前将其丢弃，或明确标记为仅内部生成字段。
